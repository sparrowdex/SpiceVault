# Spice Vault Project Explanation

## Project Overview

Spice Vault is a comprehensive full-stack web application designed to serve as a centralized platform for recipe sharing and discovery. The application aims to make cooking accessible and enjoyable by providing users with a vast database of recipes from diverse cuisines worldwide. It combines traditional recipe browsing features with advanced machine learning-powered recommendations to create a personalized cooking experience.

The platform allows users to explore recipes, contribute their own creations, rate and review dishes, and receive intelligent recommendations based on their preferences and behavior. Spice Vault targets both novice cooks looking for easy recipes and experienced chefs seeking inspiration from global cuisines.

## Tech Stack

- **Frontend**: React.js (with Hooks)
- **Build Tool**: Vite
- **Styling**: Tailwind CSS
- **Routing**: React Router
- **File Uploads**: UploadThing SDK
- **Backend**: Node.js with Express.js
- **Database**: MySQL with Prisma ORM
- **Authentication**: JWT (JSON Web Tokens) with bcrypt for password hashing
- **Machine Learning**: Custom ML service using `ml-matrix` and `natural`

## Core Features

### 1. User Authentication & Profiles
- Secure user registration and login with JWT authentication.
- User profiles displaying saved, liked, rated, and added recipes.
- Role-based access control (Regular User vs. Chef).

### 2. Recipe Management
- **Browse & Discover**: A rich homepage with an endless, auto-playing carousel for popular recipes and a filterable grid for all recipes.
- **Advanced Filtering**: Filter recipes by difficulty, food category, and diet type through an interactive carousel.
- **Search**: Full-text search for recipe titles.
- **Recipe Details**: Comprehensive recipe pages with ingredients, instructions, nutritional information, and images, all beautifully auto-formatted into lists or justified text.
- **Add Recipes (Chefs Only)**: Chefs can contribute their own recipes with a streamlined form, including a themed drag-and-drop image uploader via UploadThing.

### 3. Social & Community Features
- **Rating & Reviews**: Users can leave 1-5 star ratings and detailed reviews. The reviews section includes pagination for a clean UI.
- **Chef Replies**: Recipe authors (chefs) can reply directly to user reviews.
- **User Interactions**: Users can 'like' and 'save' recipes, which feeds into the ML engine.
- **Global Rankings**: A dynamic leaderboard page showcasing top recipes, filterable by daily, weekly, monthly, and all-time periods.

### 4. Advanced Chef Features
- **Chef Insights Dashboard**: A dedicated analytics dashboard for chefs to track their performance. It includes:
  - Key metrics (average rating, total recipes, success score).
  - A visual bar chart showing 6-month rating trends.
  - A showcase of their most popular recipes.
  - A slideshow of recent reviews on their recipes.
  - "Smart AI Tips" that provide personalized advice based on their actual stats.

## Machine Learning Implementation

Spice Vault incorporates a sophisticated machine learning recommendation system to provide personalized suggestions.

### ML Techniques

#### 1. Hybrid Recommendation System
- **Core Strategy**: The primary recommendation engine uses a hybrid approach, combining the strengths of both Collaborative and Content-Based filtering.
- **Collaborative Filtering**: Analyzes the behavior of similar users to recommend recipes that the current user might enjoy. It builds a user-item matrix and calculates user similarity using cosine similarity.
- **Content-Based Filtering**: Analyzes the properties of recipes (difficulty, category, diet type) that a user has positively interacted with to recommend similar items.
- **Score Combination**: The final recommendation score is a weighted average (60% collaborative, 40% content-based) to provide balanced and accurate suggestions.

#### 2. Nutritional Recommendations
- **Health Score**: A custom algorithm calculates a health score (0-100) for each recipe based on its nutritional information (calories, protein, fat, etc.).
- **Personalized Health Focus**: Users can request recommendations based on a health focus, such as "Excellent" or "Very Healthy", which prioritizes recipes with high health scores.

### Data & Infrastructure
- **User Interaction Tracking**: The system logs views, likes, and saves to continuously learn user preferences.
- **Cold Start Problem**: For new users or in case of ML failure, the system gracefully falls back to showing globally popular or random recipes.
- **Performance**: The ML models and matrices are updated periodically (every 24 hours) to ensure recommendations are fresh without causing real-time performance bottlenecks.

## Project Modernization

This project underwent a significant "Great Modernization Refactor" to upgrade its entire technology stack from legacy tools to modern, industry-standard alternatives.

- **From Create React App to Vite**: Drastically improved build times and developer experience.
- **From Component CSS to Tailwind CSS**: Centralized styling, eliminated CSS conflicts, and enabled rapid, responsive UI development.
- **From Sequelize to Prisma**: Migrated the entire backend data layer to a modern, type-safe ORM, simplifying database queries and migrations.
- **From Local Storage to Cloud Uploads**: Integrated UploadThing for seamless, direct-to-cloud image uploads, replacing a manual URL-pasting system.

This refactor has resulted in a more performant, maintainable, and scalable application, ready for future feature development and deployment.
